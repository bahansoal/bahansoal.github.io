12. apa perbedaan ? 1.IP Private statik 2.IP Private dinamik 3.IP publik statik 4.IP publik dinamik


Penjelasan:

IP Public adalah Internet Protocol (IP) yang bisa diakses langsung oleh internet. Jika dianalogikan, IP Public itu telepon rumah atau nomer HP yang bisa ditelepon langsung oleh semua orang.

IP Private adalah IP yang biasanya digunakan dalam jaringan yang tidak terhubung ke internet atau bisa juga terhubung ke internet tapi melalui Network Address Translation (NAT). Analoginya IP private itu telepon lokal dalam kantor/ hotel yang bisa untuk telepon-teleponan gratis dalam satu gedung.

IP Dynamic itu berarti alokasi IPnya bisa berubah-ubah. Biasa menggunakan DHCP server. Di penganturan jaringan lokal komputer kita biasanya menggunakan setting automatis.

Static IP Address/Dedicated IP Address hanya bisa dipasang untuk domain utama.

18. Pada sebuah sekolahan A memiliki 2 ruangan. 1 ruangan memiliki 4 client, ruangan ke-2 memiliki 6 client. Sekolahan tersebut akan membuat sebuah jaringan yag terhubung dengan ISP, dengan IP. 85.87.35.1. Jaringan tersebut menggunakan sebuah router yang memiliki 5 port.1. Tentukan IP port 1.2. Tentukan IP port 2 dan port 3 dengan IP clas C, serta tentukan IP masing-masing client.3. Berapa IP route pada router tersebut.​


Jawaban:

1. IP port 1 pada router dapat ditentukan dengan menggunakan IP jaringan sekolahan dan subnet mask. Dalam hal ini, IP jaringan sekolahan adalah 85.87.35.0 dengan subnet mask 255.255.255.0.

Untuk menentukan IP port 1, kita perlu menetapkan alamat IP yang valid dalam jaringan tersebut. Biasanya, IP port 1 adalah alamat IP pertama dalam jaringan. Jadi, IP port 1 adalah 85.87.35.1.

2. Untuk menentukan IP port 2 dan port 3, serta IP masing-masing client, kita perlu menggunakan IP kelas C. IP kelas C memiliki subnet mask 255.255.255.0, yang berarti hanya 8 bit terakhir yang digunakan untuk mengidentifikasi host dalam jaringan.

Dalam ruangan pertama dengan 4 client, kita dapat menggunakan alamat IP berikut:

IP port 2: 85.87.35.2

IP client 1: 85.87.35.3

IP client 2: 85.87.35.4

IP client 3: 85.87.35.5

IP client 4: 85.87.35.6

Dalam ruangan kedua dengan 6 client, kita dapat menggunakan alamat IP berikut:

IP port 3: 85.87.35.7

IP client 1: 85.87.35.8

IP client 2: 85.87.35.9

IP client 3: 85.87.35.10

IP client 4: 85.87.35.11

IP client 5: 85.87.35.12

IP client 6: 85.87.35.13

3. Router tersebut memiliki 5 port, sehingga kita perlu menentukan berapa banyak IP route yang tersedia. Dalam hal ini, kita menggunakan IP kelas C dengan subnet mask 255.255.255.0.

Dalam subnet mask 255.255.255.0, hanya 8 bit terakhir yang digunakan untuk mengidentifikasi host. Oleh karena itu, kita memiliki 2^8 - 2 = 254 IP host yang dapat digunakan dalam jaringan tersebut. Dua IP terakhir (85.87.35.0 dan 85.87.35.255) tidak dapat digunakan sebagai IP host, sehingga kita memiliki 254 - 2 = 252 IP host yang tersedia.Namun, router tidak menggunakan IP host dalam jaringan tersebut. Sebagai gantinya, router menggunakan IP route. Jadi, jumlah IP route pada router tersebut adalah 252.

Jawaban:

1.) 192.168.1.15/24

a. Jumlah Subnet = 2^(jumlah bit subnet) = 2^0 = 1

b. Jumlah host per subnet = 2^(jumlah bit host) - 2 = 2^8 - 2 = 254

c. Blok subnet = 256 - subnet mask oktet terakhir = 256 - 255 = 1

d. Subnet ke-0 dari 1

e. IP Network = IP Address AND Subnet Mask = 192.168.1.15 AND 255.255.255.0 = 192.168.1.0

f. IP Host awal = IP Network + 1 = 192.168.1.0 + 1 = 192.168.1.1

g. IP Host akhir = IP Broadcast - 1 = 192.168.1.255 - 1 = 192.168.1.254

h. IP Broadcast = IP Network OR (NOT Subnet Mask) = 192.168.1.0 OR (NOT 255.255.255.0) = 192.168.1.255

2.) 192.168.2.115/24

a. Jumlah Subnet = 2^(jumlah bit subnet) = 2^0 = 1

b. Jumlah host per subnet = 2^(jumlah bit host) - 2 = 2^8 - 2 = 254

c. Blok subnet = 256 - subnet mask oktet terakhir = 256 - 255 = 1

d. Subnet ke-0 dari 1

e. IP Network = IP Address AND Subnet Mask = 192.168.2.115 AND 255.255.255.0 = 192.168.2.0

f. IP Host awal = IP Network + 1 = 192.168.2.0 + 1 = 192.168.2.1

g.IP Host akhir = IP Broadcast - 1 = 192 .168 .2 .255 - 1=192 .168 .2 .254

h.IP Broadcast=IP Network OR (NOT Subnet Mask)=192 .168 .2 .0 OR (NOT255 .255 .255 .0)=192 .168 .2 .255

c.Blok subnet=256-subnet mask oktet terakhir=256-252=4

d.Subnet ke-3 dari64

f.IP Host awal=IP Network+1=192 .168 .3 .12+1=192 .168 .3 .13

g.IP Host akhir=IP Broadcast-1=192 .168 .3 .15-1=192 .168 .3 .14

34. 2. ip address 100.100.100.100tentukan class :oktet network:oktet host:subner mask:ip network:ip host:ip bc :​


Alamat IP 100.100.100.100 adalah contoh alamat IP, dan kita dapat menganalisisnya sebagai berikut:

Class: Alamat IP ini adalah bagian dari kelas A, karena rentang alamat IP kelas A adalah 0.0.0.0 hingga 127.255.255.255.

Oktet Network: Pada alamat IP kelas A, oktet pertama (100) adalah bagian dari alamat jaringan.

Oktet Host: Tiga oktet berikutnya (100.100.100) adalah bagian dari alamat host.

Subnet Mask: Karena ini adalah alamat kelas A, subnet mask standarnya adalah 255.0.0.0.

IP Network: Alamat jaringan adalah 100.0.0.0 (dengan tiga oktet pertama dari alamat IP).

IP Host: Alamat host adalah 0.100.100.100 (dengan tiga oktet terakhir dari alamat IP).

IP Broadcast: Alamat broadcast dalam jaringan ini adalah 100.255.255.255 (dengan tiga oktet pertama tetap dan oktet terakhir diatur menjadi 255).

40. Soal 1. Carilah IP di masing-masing komputer a. 150 komputer 6.30 komputer C. 10 komputer d. 2 komputer dengan IP AWAL : 200.172.10.10


Jawaban:

Untuk mencari alamat IP pada masing-masing komputer, bisa dilakukan dengan beberapa cara, antara lain:

Pada sistem operasi Windows, buka Command Prompt dan ketikkan "ipconfig" (tanpa tanda petik), kemudian tekan enter. Alamat IP akan terlihat pada bagian "IPv4 Address".

Pada sistem operasi Mac, buka Terminal dan ketikkan "ifconfig" (tanpa tanda petik), kemudian tekan enter. Alamat IP akan terlihat pada bagian "inet".

Setelah mengetahui alamat IP, Anda dapat menggunakan pola alamat IP yang sama dan menambahkan angka pada bagian terakhirnya untuk mencari alamat IP pada komputer lain. Misalnya, jika IP awal adalah 200.172.10.10, maka IP pada komputer berikutnya adalah 200.172.10.11, 200.172.10.12, dst.

Untuk komputer A (150 komputer), bisa ditemukan dengan menambahkan angka dari 0 hingga 149 pada bagian terakhir alamat IP.

Untuk komputer B (6.30 komputer), bisa ditemukan dengan menambahkan angka dari 0 hingga 29 pada bagian terakhir alamat IP.

Untuk komputer C (10 komputer), bisa ditemukan dengan menambahkan angka dari 0 hingga 9 pada bagian terakhir alamat IP.

Untuk komputer D (2 komputer), bisa ditemukan dengan menambahkan angka dari 0 hingga 1 pada bagian terakhir alamat IP.
