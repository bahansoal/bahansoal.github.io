Diketahui IP Address PC Server yaitu 192.168.1.1/25, tersambungke 1 buah router A dengan IP Address 192.168.25.1, Router A tersambungdengan Router B yang mempunyai IP Address 192.168.26.1, Switch A, dan Access Point A. Router A, Switch A dan Access Point A tersambung masing-masing 1 PC 1. Berapakah IP Address PC Yang tersambungoleh Switch A... a. 192.168.1.2 b. 192.168.100.1 C. 192.168.25.10 d. 192.168.1.10 192.168.100.2 lah In Addros RC Yang tersambungoleh Router B e. O​


Untuk menentukan IP Address PC yang tersambung oleh Switch A, kita perlu melihat subnet yang digunakan oleh perangkat dalam jaringan ini. Berdasarkan informasi yang diberikan, kita tahu bahwa IP Address PC Server adalah 192.168.1.1/25. Notasi /25 menunjukkan bahwa subnet mask adalah 255.255.255.128.

Kemudian, kita tahu bahwa Router A memiliki IP Address 192.168.25.1. Ini menunjukkan bahwa Router A berada dalam subnet yang berbeda dari PC Server. Router A, Switch A, dan Access Point A masing-masing tersambung ke 1 PC. Dikarenakan Switch A berada dalam jaringan yang sama dengan Router A, kita bisa berasumsi bahwa PC yang tersambung ke Switch A memiliki IP Address dalam subnet yang sama dengan Router A.

Dari pilihan yang diberikan, hanya IP Address 192.168.25.10 (pilihan C) yang berada dalam subnet yang sama dengan Router A (192.168.25.1). Oleh karena itu, IP Address PC yang tersambung oleh Switch A adalah 192.168.25.10.

4. 5. Diketahui router dalam jaringan mempunyai ip address 120.123.98.78/29, ip network dari router tersebut adalah.... 1. 120.123.98.0 2. 120.123.98.56 3. 120.123.98.64 4. 120.123.98.72 5. 120.123.98.88​


Jawaban:

4). 120.123.98.72

Penjelasan:

Alamat jaringan dari router dengan IP address 120.123.98.78/29 adalah 120.123.98.72. Subnet mask /29 mengindikasikan bahwa 3 bit pertama dari alamat IP digunakan untuk jaringan, sehingga 8 - 3 = 5 bit tersisa untuk host. Dengan demikian, alamat jaringan adalah hasil dari penghapusan 5 bit terakhir dari IP address, yaitu 78 - 6 = 72. Sehingga, alamat jaringannya adalah 120.123.98.72.

13. Bagaimana caranya agar setiap PC/Client mendapatkan alamat IP secara otomatis dari Router melalui CLI?​


Jawaban:

Dynamic Host Configuration Protocol (DHCP) merupakan service yang memungkinkan perangkat dapat mendistribusikan/assign IP Address secara otomatis pada host dalam sebuah jaringan. Cara kerjanya, DHCP Server akan memberikan response terhadap request yang dikirimkan oleh DHCP Client.

Selain IP Address, DHCP juga mampu mendistribusikan informasi netmask, Default gateway, Konfigurasi DNS dan NTP Server serta masih banyak lagi custom option (tergantung apakah DHCP client bisa support).

Mikrotik dapat digunakan sebagai DHCP Server maupun DHCP Client atau keduanya secara bersamaan. Sebagai contoh, misalnya kita berlangganan internet dari ISP A. ISP A tidak memberikan informasi IP statik yang harus dipasang pada perangkat kita, melainkan akan memberikan IP secara otomatis melalui proses DHCP.

20. 2. Tabel routing adalah tabel yang memuat seluruh informasi IP address dari interface router yang lain sehingga router yang satu dengan router yang lainnya bisa berkomunikasi. Sebut dan jelaskan 5 (lima) macam tabel routing yang kamu ketahui.​


Jawaban:

Tabel Routing Statis:

Tabel ini diatur secara manual oleh administrator jaringan.

Setiap entri di tabel ini harus ditambahkan atau diubah secara manual oleh administrator.

Cocok untuk jaringan kecil atau konfigurasi sederhana.

Tabel Routing Dinamis:

Tabel ini diperbarui secara otomatis oleh protokol routing.

Router bertukar informasi untuk memahami topologi jaringan dan memperbarui tabel sesuai keadaan jaringan.

Cocok untuk jaringan yang kompleks dan dinamis.

Tabel Forwarding:

Tabel ini digunakan untuk menentukan jalur mana yang harus diambil oleh paket berdasarkan alamat tujuan.

Biasanya diperbarui dari tabel routing, dan digunakan secara aktif dalam pengiriman paket.

Tabel ARP (Address Resolution Protocol):

Mengaitkan alamat IP dengan alamat fisik (MAC address) di jaringan lokal.

Digunakan untuk mengirim paket dalam jaringan lokal.

Tabel NAT (Network Address Translation):

Mengonversi alamat IP internal menjadi alamat IP eksternal, memungkinkan beberapa perangkat di jaringan internal menggunakan satu alamat IP eksternal.

Penting untuk mengatasi kelangkaan alamat IP publik.

Tabel-tabel ini membantu router untuk membuat keputusan tentang bagaimana mengarahkan paket data melalui jaringan.

Sekolahan tersebut akan membuat sebuah jaringan yag terhubung dengan ISP, dengan IP. 85.87.35.1. Jaringan tersebut menggunakan sebuah router yang memiliki 5 port.1. Tentukan IP port 1.2. Tentukan IP port 2 dan port 3 dengan IP clas C, serta tentukan IP masing-masing client.3. Berapa IP route pada router tersebut.​


Jawaban:

1. IP port 1 pada router dapat ditentukan dengan menggunakan IP jaringan sekolahan dan subnet mask. Dalam hal ini, IP jaringan sekolahan adalah 85.87.35.0 dengan subnet mask 255.255.255.0.

Untuk menentukan IP port 1, kita perlu menetapkan alamat IP yang valid dalam jaringan tersebut. Biasanya, IP port 1 adalah alamat IP pertama dalam jaringan. Jadi, IP port 1 adalah 85.87.35.1.

2. Untuk menentukan IP port 2 dan port 3, serta IP masing-masing client, kita perlu menggunakan IP kelas C. IP kelas C memiliki subnet mask 255.255.255.0, yang berarti hanya 8 bit terakhir yang digunakan untuk mengidentifikasi host dalam jaringan.

Dalam ruangan pertama dengan 4 client, kita dapat menggunakan alamat IP berikut:

IP port 2: 85.87.35.2

IP client 1: 85.87.35.3

IP client 2: 85.87.35.4

IP client 3: 85.87.35.5

IP client 4: 85.87.35.6

Dalam ruangan kedua dengan 6 client, kita dapat menggunakan alamat IP berikut:

IP port 3: 85.87.35.7

IP client 1: 85.87.35.8

IP client 2: 85.87.35.9

IP client 3: 85.87.35.10

IP client 4: 85.87.35.11

IP client 5: 85.87.35.12

IP client 6: 85.87.35.13

3. Router tersebut memiliki 5 port, sehingga kita perlu menentukan berapa banyak IP route yang tersedia. Dalam hal ini, kita menggunakan IP kelas C dengan subnet mask 255.255.255.0.

Dalam subnet mask 255.255.255.0, hanya 8 bit terakhir yang digunakan untuk mengidentifikasi host. Oleh karena itu, kita memiliki 2^8 - 2 = 254 IP host yang dapat digunakan dalam jaringan tersebut. Dua IP terakhir (85.87.35.0 dan 85.87.35.255) tidak dapat digunakan sebagai IP host, sehingga kita memiliki 254 - 2 = 252 IP host yang tersedia.Namun, router tidak menggunakan IP host dalam jaringan tersebut. Sebagai gantinya, router menggunakan IP route. Jadi, jumlah IP route pada router tersebut adalah 252.

1. Ganti IP Address yang terindikasi Conflict IP (Tabrakan IP):

Terdapat konflik IP pada jaringan antara PC 01 dan PC 02, karena keduanya memiliki alamat IP yang sama (192.168.1.1). Kita perlu mengganti salah satu alamat IP agar tidak terjadi konflik. Misalnya, kita ganti IP PC 02 menjadi 192.168.1.3.

Setelah perubahan, topologi jaringan menjadi:

```

+------------+

| PC 01 |

|192.168.1.1|

+------------+

|

|

|

+------------+

| PC 02 |

|192.168.1.3|

+------------+

|

|

|

+------------+

| PC 03 |

|192.168.2.3|

+------------+

```

2. Tentukan kelas IP Address pada topologi tersebut:

Berdasarkan alamat IP yang diberikan, kita dapat melihat bahwa semua alamat IP dalam topologi ini termasuk dalam kelas C IP Address, karena 3 oktet pertama dari alamat IP tetap sama untuk semua perangkat (192.168.1.x atau 192.168.2.x).

3. Tentukan Default Gateway dan Subnet Mask yang cocok untuk konfigurasi IP topologi di atas:

Dalam topologi ini, kita tidak memiliki informasi tentang gateway atau subnet mask yang diperlukan. Default gateway adalah alamat IP perangkat yang digunakan untuk menghubungkan jaringan lokal dengan jaringan luar, dan subnet mask digunakan untuk memisahkan alamat IP menjadi bagian jaringan dan bagian host.

Tanpa informasi tambahan, tidak mungkin menentukan default gateway atau subnet mask yang tepat untuk topologi ini. Biasanya, default gateway akan bergantung pada perangkat jaringan seperti router atau switch yang menghubungkan jaringan lokal dengan jaringan luar.

Subnet mask akan tergantung pada desain jaringan yang diinginkan, misalnya, jika kita ingin memisahkan PC 01 dan PC 02 dalam satu jaringan tetapi memisahkan PC 03 dalam jaringan terpisah, maka kita akan menggunakan subnet mask yang memungkinkan pengelompokan IP sesuai dengan itu.

Dengan informasi yang diberikan, kita tidak dapat menentukan default gateway atau subnet mask yang cocok tanpa informasi tambahan tentang desain jaringan yang diinginkan.
